# 互動式計畫書生成器 - 問答流程說明

## 📋 問題結構

總共 **25 個問題**，分為 7 大類別：

### 1. 基本資訊（5 題）
- 公司名稱
- 產業別
- 員工人數
- 實收資本額
- 計畫主持人

### 2. 問題陳述（5 題）
- 核心問題描述
- 問題嚴重程度
- 現有解決方案的缺點
- 客戶驗證（訪談數量）
- 客戶痛點評分

### 3. 創新構想（5 題）
- 解決方案描述
- 創新點列舉
- 競爭優勢
- 量化效益
- 技術門檻

### 4. 市場分析（3 題）
- 目標市場
- 市場規模（TAM/SAM/SOM）
- 商業模式

### 5. 可行性評估（3 題）
- 目前 TRL
- 目標 TRL
- 主要風險與對策

### 6. 團隊組成（2 題）
- 核心團隊成員
- 團隊經驗

### 7. 經費規劃（2 題）
- 總經費
- 經費分配

---

## 🎯 問題類型

### text（文字輸入）
單行文字，如公司名稱

### textarea（多行文字）
多行文字，如問題描述、解決方案

### number（數字）
數字輸入，如員工人數、經費

### choice（選擇題）
單選，如產業別、商業模式

### scale（評分）
1-10 分評分，如問題嚴重程度

---

## ✅ 驗證規則

### 必填檢查
- `required: true` 的問題必須回答

### 長度限制
- `min_length` / `max_length`：文字長度限制
- 確保回答足夠詳細但不冗長

### 數值範圍
- `min` / `max`：數字範圍限制
- 超過範圍會顯示警告

### 條件顯示
- `depends_on`：根據前面的答案決定是否顯示
- 例如：只有訪談過客戶才問痛點評分

## 🔄 使用流程

### Pro 用戶（Claude Projects）

1. 創建 Project：「我的 SBIR 計畫書」
2. 上傳 `questions.json` 到 Project Knowledge
3. 設定 Custom Instructions（見 `project_instructions.md`）
4. 開始對話：「開始生成 Phase 1 計畫書」
5. Claude 逐題詢問，Projects 自動記住答案
6. 完成後自動生成計畫書

### 免費用戶（MCP Server）

1. 開始對話：「開始生成 Phase 1 計畫書」
2. Claude 呼叫 MCP Server 的 `start_proposal_generator`
3. MCP Server 載入 `questions.json`
4. 逐題詢問，每次回答後呼叫 `save_answer`
5. 可以中斷，下次呼叫 `continue_proposal_generator` 繼續
6. 完成後呼叫 `generate_proposal` 生成計畫書

## 💡 設計原則

### 1. 循序漸進
從簡單的基本資訊開始，逐步深入

### 2. 邏輯分組
相關問題放在一起，便於思考

### 3. 適度詳細
既要收集足夠資訊，又不能太繁瑣

### 4. 彈性設計
非必填問題可以跳過，之後補充

### 5. 即時驗證
輸入時就檢查，避免最後才發現錯誤
