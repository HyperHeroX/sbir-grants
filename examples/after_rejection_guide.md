# 失敗後的下一步

> 未通過不是結束，而是重新出發的機會。

## 正確看待未通過

### 心態調整

**事實**：
- SBIR 整體通過率約 **30-40%**
- 首次申請通過率更低
- 許多成功企業都經歷過失敗

**意義**：
- 未通過 ≠ 技術不好
- 可能是呈現方式問題
- 可能是策略問題
- 有機會修正再來

---

## 解讀審查意見

### 取得審查意見

**如何取得**：
1. 等候計畫辦公室通知
2. 主動洽詢計畫辦公室
3. 通常會提供「審查委員意見摘要」

### 審查意見類型

| 類型 | 說明 | 處理方式 |
|------|------|----------|
| 技術疑慮 | 可行性不足 | 補充數據/調整方法 |
| 創新不足 | 差異化不明 | 重新定位/強化創新 |
| 市場問題 | 商業化不清 | 強化市場分析 |
| 團隊疑慮 | 能力不足 | 補強團隊/找顧問 |
| 經費問題 | 編列不合理 | 調整預算 |

### 解讀範例

**審查意見**：
> 「本計畫創新點不夠明確，與市面現有產品差異不大。」

**解讀**：
- 問題：創新性不足
- 原因可能：
  1. 確實創新不夠
  2. 創新點沒有清楚說明
  3. 競品分析不完整

**行動**：
- 重新檢視技術差異
- 改寫創新說明
- 加強競品比較

---

## 常見未通過原因分析

### 原因一：創新性不足

**症狀**：
- 「與現有技術差異不大」
- 「未見明顯創新」

**可能問題**：
1. 技術本身創新有限
2. 沒有說清楚創新在哪
3. 沒有突顯技術優勢

**解決方案**：

| 問題 | 解法 |
|------|------|
| 創新有限 | 找新切入點或調整技術路線 |
| 說不清楚 | 用對比表/數據呈現差異 |
| 沒突顯 | 重新組織內容，強調亮點 |

---

### 原因二：技術可行性不足

**症狀**：
- 「缺乏前期驗證」
- 「技術風險過高」

**可能問題**：
1. 沒有前期實驗數據
2. 技術成熟度太低
3. 沒有說明如何解決風險

**解決方案**：

| 問題 | 解法 |
|------|------|
| 沒有數據 | 先做小規模驗證 |
| 成熟度低 | 先申請 Phase 1 |
| 風險說明 | 加入風險分析與對策 |

---

### 原因三：市場分析不足

**症狀**：
- 「商業模式不清楚」
- 「市場規模評估不合理」

**可能問題**：
1. TAM/SAM/SOM 計算有誤
2. 沒有說明誰付錢
3. 競爭分析不完整

**解決方案**：

| 問題 | 解法 |
|------|------|
| 規模錯誤 | 引用可靠數據來源 |
| 付費者不清 | 說明商業模式 |
| 競爭不明 | 完整競品分析 |

---

### 原因四：團隊能力不足

**症狀**：
- 「團隊缺乏相關經驗」
- 「執行能力存疑」

**可能問題**：
1. 團隊真的缺乏經驗
2. 經驗沒有有效呈現
3. 少了關鍵角色

**解決方案**：

| 問題 | 解法 |
|------|------|
| 缺乏經驗 | 補充團隊成員/顧問 |
| 沒呈現 | 重寫團隊介紹 |
| 缺角色 | 找產學合作/技術顧問 |

---

### 原因五：計畫書品質問題

**症狀**：
- 「內容組織不佳」
- 「表達不清楚」

**可能問題**：
1. 邏輯結構混亂
2. 文字冗長不精練
3. 圖表使用不當

**解決方案**：

| 問題 | 解法 |
|------|------|
| 結構差 | 重新組織章節 |
| 太冗長 | 精簡，用重點列表 |
| 圖表少 | 增加視覺化呈現 |

---

## 修正策略

### 方法一：針對意見修正

```
審查意見 → 分析問題 → 擬定對策 → 修正內容 → 再次送件
```

**適用情況**：
- 審查意見明確
- 問題可以解決
- 核心構想穩固

### 方法二：調整計畫方向

**適用情況**：
- 原構想有根本問題
- 市場/技術路線需改變
- 需要重新定位

**做法**：
1. 重新評估市場需求
2. 調整技術方向
3. 重寫計畫書

### 方法三：先累積再申請

**適用情況**：
- 前期驗證不足
- 團隊能力待補強
- 需要更多時間準備

**做法**：
1. 三個月內完成小規模驗證
2. 補強團隊或找顧問
3. 準備更充分後再申請

---

## 再次申請注意事項

### 時間規劃

```
收到審查意見
      ↓
分析問題（1週）
      ↓
擬定對策（1週）
      ↓
修正/補強（1-2月）
      ↓
重寫計畫書（2-4週）
      ↓
下一梯次送件
```

### 修正重點

| 項目 | 注意事項 |
|------|----------|
| 針對意見 | 明確回應審查意見 |
| 補充數據 | 加入新的驗證成果 |
| 強化亮點 | 重新組織突顯優勢 |
| 精練表達 | 文字精簡、圖表增加 |
| 找人審閱 | 請他人給予回饋 |

### 不要只改表面

**❌ 錯誤做法**：
- 只修改文字措辭
- 沒有補充新資料
- 沒有真正解決問題

**✅ 正確做法**：
- 針對問題做實質改善
- 補充新的驗證數據
- 重新思考策略定位

## 成功翻身案例

### 案例一：從 Phase 1 失敗到 Phase 2 通過

**第一次（失敗）**：
- 問題：創新說明不清、缺數據
- 分數：65 分（差 5 分通過）

**分析與改善**：
1. 加入 3 個月實測數據
2. 請專家顧問協助撰寫
3. 重新設計簡報

**第二次（失敗）**：
- 問題：市場分析不足

**分析與改善**：
1. 補充完整競品分析
2. 加入客戶訪談結果
3. 強化商業模式

**第三次（通過）**：
- 通過 Phase 1
- 後續成功申請 Phase 2

**關鍵**：每次都針對問題做實質改善

---

### 案例二：調整方向後成功

**第一次**：
- 計畫：開發全新平台
- 結果：失敗（技術風險太高）

**調整**：
- 改為：針對特定產業的解決方案
- 縮小範圍，降低風險

**第二次**：
- 結果：成功通過
- 原因：定位更精準，可行性更高

## 心理建設

### 保持積極

- 失敗是學習的機會
- 每次申請都是練習
- 成功的企業都經歷過挫折

### 尋求支持

- 與有經驗的人交流
- 參加 SBIR 說明會
- 善用政府輔導資源

### 設定合理期望

- 首次申請通過率低於 30%
- 準備好多次嘗試
- 持續改進終將成功
